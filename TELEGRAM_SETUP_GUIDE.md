# 🤖 11원칙 퀀트 에이전트: 텔레그램 자동 알림 봇 세팅 가이드

본 문서에서는 깃허브 액션(GitHub Actions)을 활용해 **매일 자동으로 주식 시장 진바닥 점수와 타겟 종목(AVGO) 매수 시그널을 계산하고, 그 결과를 텔레그램으로 전송**하는 시스템에 대해 설명합니다. 다른 컴퓨터에서 이어서 작업하거나 재설정이 필요할 때 참고하세요.

---

## 1. 시스템 구조 및 포함된 파일
* **`alert_bot.py`**: 퀀트 점수(미국/한국) 및 타겟 종목 시그널을 계산하고, 텔레그램 API를 호출하여 메시지를 발송하는 핵심 파이썬 스크립트입니다.
* **`.github/workflows/alert.yml`**: 깃허브 서버(클라우드)에서 봇을 언제, 어떻게 실행할지 정의한 스케줄러 파일입니다.
  * **자동 실행 스케줄**: 매일 미장 마감 직후(한국시간 오전 6시) & 한국장 마감 직전(한국시간 오후 3시 20분)
  * **수동 실행 지원**: 깃허브 페이지에서 `workflow_dispatch` 버튼을 눌러 언제든 즉시 실행 가능합니다.

---

## 2. 깃허브 시크릿(Secrets) 설정 방법
이 코드가 작동하려면 깃허브 저장소에 텔레그램 봇의 인증 정보가 안전하게 저장되어 있어야 합니다.

1. 깃허브 저장소 상단 메뉴의 **[Settings]** 클릭
2. 좌측 메뉴의 **[Secrets and variables]** -> **[Actions]** 클릭
3. **[New repository secret]** 버튼을 눌러 아래 두 가지를 생성합니다.
   * **`TELEGRAM_BOT_TOKEN`**: BotFather에게 받은 봇 토큰 (예: `8968787593:AAExeSuAqUX...`)
   * **`TELEGRAM_CHAT_ID`**: 텔레그램에서 메시지를 받을 대장님의 개인 채팅방 ID (숫자, 예: `8332273741`)

---

## 3. 🚨 [필수] 텔레그램 봇 활성화 (에러 해결법)
토큰과 ID가 정확하더라도, 텔레그램 정책상 **사용자가 먼저 봇에게 말을 걸지 않으면 스팸으로 간주되어 알림 발송이 차단**됩니다. (이 경우 깃허브 로그에 `400 Bad Request: chat not found` 에러가 발생합니다.)

**[해결 방법: 봇 빗장 풀기]**
1. 스마트폰이나 PC에서 텔레그램 앱 실행
2. 상단 돋보기(검색) 아이콘을 눌러 본인의 봇 아이디 검색 (예: **`@Junitoniii_bot`**)
3. 검색 결과에서 봇(이름: Jun)을 클릭하여 입장
4. 화면 맨 아래에 있는 **[시작(Start)]** 버튼을 클릭하거나, 채팅창에 `안녕` 이라고 쳐서 전송!
5. 이제 봇이 대장님께 알림을 보낼 수 있는 권한을 얻게 됩니다.

---

## 4. 텔레그램 알림 수동 발사 테스트
다른 컴퓨터에서 세팅을 마쳤거나, 봇이 잘 작동하는지 즉시 확인하고 싶을 때 사용합니다.

1. 깃허브 저장소 상단 메뉴의 **[Actions]** 탭 클릭
2. 좌측 메뉴(Workflows)에서 **`Quant Alert Bot`** 클릭
3. 우측의 **[Run workflow ▾]** 버튼을 누르고 초록색 **[Run workflow]** 클릭
4. 노란색 팽이 아이콘(`In progress`)이 돌면서 서버가 기동됩니다 (보통 1분~1분 30초 소요).
5. 초록색 체크표시(`✅`)로 바뀌면, 스마트폰 텔레그램으로 "🔔 [11원칙 퀀트 에이전트 정기 보고]" 알림이 도착합니다.

> **참고**: 봇은 기본적으로 시장 점수가 80점 미만이면 알림을 생략하도록 설계되어 있으나, 현재 `alert_bot.py` 코드는 **연결 테스트를 위해 점수에 상관없이 무조건 요약 상태 보고를 전송**하도록 임시 설정되어 있습니다. 추후 점수 미달 시 알림을 끄고 싶다면 `alert_bot.py` 하단의 로직을 수정하면 됩니다.
