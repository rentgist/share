# 11원칙 퀀트 대시보드 v27.9

## 🛠️ 최근 업데이트 내역 (2026-07-15)
- **상승장 눌림목 GTC 타점 스마트 하향 조정**: 상승장에서 눌림목이 발생했을 때, 주가가 20일 이평선(1차 지지선)을 이미 하향 돌파했다면 볼린저 밴드 하단으로 GTC(지정가 매수) 타점을 자동 하향 조정하여 떨어지는 칼날을 피하도록 알고리즘을 개선했습니다.
- **글로벌 뉴스 타임라인 72시간 필터링**: 수집된 주요 뉴스 아카이브에서 72시간이 경과한 철 지난 폭락/폭등 이슈는 AI 분석 및 웹 UI에서 자동으로 배제하여 시의성을 대폭 끌어올렸습니다.
- **관찰 기간 영구 상태(Persistent State) 저장**: 세션 휘발성 문제로 앱 재시작 시 초기화되던 경고 상태(1~5일 관찰 중)를 `regime_state.json`에 안전하게 보관하여 모바일/PC 연동을 완벽히 지원합니다.
- **비중 조절(Position Sizing) 룰 박제**: 선발대(10~20%)와 본대(50%) 투입 원칙 및 강력 GO 구간에서의 눌림목 매수 원칙을 대시보드 스나이퍼 탭 및 레이더 탭 전면에 영구 박제했습니다.

## 🛠️ 이전 업데이트 내역 (2026-07-14)

  - 8대 핵심 타겟 종목(삼성전자, SK하이닉스, 브로드컴 등) 지정 스캔 및 텐배거 'Must-Enter' (실적 우량 + 고점대비 -20% 이상 하락 등) 전수 스캔 로직을 추가했습니다.
  - AI 분석 프롬프트에 바로 붙여넣을 수 있는 상세 데이터와 요약본을 분리하여 Two-Track으로 전송합니다.
  - **[NEW]** 개별 종목 알림 발송 시, 전체 시장의 현재 상태(미국/한국 진바닥 점수 및 국면)를 메시지 최상단에 항상 포함시켜 AI가 숲(거시경제)과 나무(개별종목)를 동시에 보고 판단할 수 있도록 개선했습니다.
- **UI 탭(Tab) 레이아웃 꼬임 버그 해결**: `final.py` 내부의 탭 생성 및 렌더링 코드 순서가 어긋나 다른 탭의 내용이 텐배거 레이더 탭에 겹쳐 출력되던 현상을 순차적 리팩토링을 통해 완벽히 해결했습니다.
- **데이터 로더(yfinance) 429 에러 방어**: `tk.info` 호출 실패 시 재무제표 원본(financials, balance_sheet 등)을 긁어와 필수 퀀트 지표를 수동으로 계산해내는 2차 백업 로직을 추가했습니다.
- **한국장(KOSPI) 백테스트 추가**: 10년치 한국장 데이터를 기반으로 한 바닥 탐지 백테스트 기능을 새롭게 추가하고, 기존 미국장 백테스트와 탭(Tab)으로 구분하여 비교할 수 있도록 개선했습니다.
- **매수 비중 전략 기준 표준화**: 투자 대가들의 '안전마진' 철학에 맞춰, 점수 대역별 1차 선발대 투입 비중을 전략적으로 일관성 있게 표준화했습니다 (20~30% ➡ 30~50% ➡ 50% 이상).
